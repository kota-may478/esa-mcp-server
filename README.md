# esa MCP Server

An MCP (Model Context Protocol) server that lets you create, edit, search, and manage esa posts directly from Claude — no browser required.

Two usage modes are supported:

| Mode | Client | Transport |
|---|---|---|
| **Local** | Claude Desktop (desktop app) | stdio (process communication) |
| **Remote** | Claude (browser — claude.ai) | HTTP (Streamable HTTP on Vercel) |

## What You Can Do

| Tool | Description |
|---|---|
| `esa_list_posts` | List and search posts (filter by category, keyword, tag, WIP status, etc.) |
| `esa_get_post` | Retrieve a specific post by post number |
| `esa_create_post` | Create a new post (with optional category, tags, WIP flag, and change message) |
| `esa_update_post` | Update an existing post |
| `esa_delete_post` | Delete a post |
| `esa_get_members` | List all team members |

## Architecture

### Local Mode (Claude Desktop)

```
You (type in chat)
      │
      ▼
┌──────────────────────────┐
│      Claude Desktop      │  AI assistant app on your PC
│    (Claude AI model)     │  understands your message and
│                          │  decides which tool to call
└────────────┬─────────────┘
             │ MCP protocol (stdio)
             │ launches and talks to ↓
             ▼
┌──────────────────────────┐
│      esa MCP Server      │  ◀──  this repository (src/index.ts)
│      dist/index.js       │  receives the tool call,
│    running on Node.js    │  calls the esa API, returns results
└────────────┬─────────────┘
             │ HTTPS  (esa REST API)
             ▼
┌──────────────────────────┐
│      Your esa Team       │  xxx.esa.io
└──────────────────────────┘
```

### Remote Mode (Browser Claude via Vercel)

```
You (type in chat at claude.ai)
      │
      ▼
┌──────────────────────────┐
│     Claude (browser)     │  claude.ai
│    (Claude AI model)     │  understands your message and
│                          │  decides which tool to call
└────────────┬─────────────┘
             │ MCP Streamable HTTP
             │ Authorization: Bearer <MCP_AUTH_TOKEN>
             ▼
┌──────────────────────────┐
│   Vercel Serverless Fn   │  ◀──  this repository (api/mcp.ts)
│  https://xxx.vercel.app  │  verifies auth token,
│       /api/mcp           │  calls the esa API, returns results
└────────────┬─────────────┘
             │ HTTPS  (esa REST API)
             ▼
┌──────────────────────────┐
│      Your esa Team       │  xxx.esa.io
└──────────────────────────┘
```

**Flow example — "Show me the 5 most recent posts":**

1. You type the message in Claude
2. Claude (AI) understands the intent and calls the `esa_list_posts` tool
3. The MCP server receives the call and sends a GET request to the esa API
4. esa returns the post data as JSON
5. The MCP server passes it back to Claude
6. Claude formats and displays it as a readable response

### Role of Each File

```
esa-mcp-server/
│
├── src/
│   ├── server.ts       ← Shared MCP server factory (all tool definitions
│   │                      and esa API calls). Used by both modes.
│   └── index.ts        ← stdio entry point for Claude Desktop.
│                          Reads env vars, creates the server, connects
│                          via StdioServerTransport.
│
├── api/
│   └── mcp.ts          ← Vercel serverless function for browser Claude.
│                          Handles HTTP requests, checks Bearer token auth,
│                          creates the server, connects via
│                          StreamableHTTPServerTransport.
│
├── dist/               ← Compiled output (auto-generated by npm run build)
│   ├── server.js          Do not edit these files directly.
│   └── index.js           Claude Desktop runs dist/index.js.
│
├── vercel.json         ← Vercel deployment configuration
├── package.json        ← Project metadata and npm script definitions
├── tsconfig.json       ← TypeScript compiler settings
└── .gitignore          ← Excludes node_modules/ and dist/ from git
```

### What Are `node_modules/` and `dist/`?

These two directories are **not stored in the repository** and must be generated locally:

| Directory | What it contains | How to generate |
|---|---|---|
| `node_modules/` | Libraries this project depends on (MCP SDK, etc.) | `npm install` |
| `dist/` | The compiled JavaScript file Claude Desktop runs | `npm run build` |

## Setup

### Step 1 — Get an esa Access Token

1. Log in to esa and open `https://[your-team-name].esa.io/user/applications`
2. Under **Personal Access Tokens**, click **"Generate new token"**
3. Select both **`read`** and **`write`** scopes, then generate the token
4. Copy and save the token somewhere safe — it is shown only once

> **Token type:** Use **Personal Access Token v1 (PAT v1)**.

> **Team name:** The team name is the `xxx` part of `xxx.esa.io`.


### Step 5 — Edit the Claude Desktop Configuration File

**File location:**
- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

**To open on Windows (PowerShell):**
```powershell
notepad "$env:APPDATA\Claude\claude_desktop_config.json"
```

Add the following to the file (replace `YOUR_*` with your actual values):

```json
{
  "mcpServers": {
    "esa": {
      "command": "node",
      "args": [
        "/absolute/path/to/esa-mcp-server/dist/index.js"
      ],
      "env": {
        "ESA_ACCESS_TOKEN": "YOUR_ESA_ACCESS_TOKEN",
        "ESA_TEAM_NAME": "YOUR_TEAM_NAME"
      }
    }
  }
}
```

**Path examples:**

macOS:
```json
"/Users/yourname/projects/esa-mcp-server/dist/index.js"
```

Windows:
```json
"C:\\Users\\yourname\\projects\\esa-mcp-server\\dist\\index.js"
```

> **Always use absolute paths.** Relative paths and `~` are not supported.

> On Windows, backslashes in JSON must be escaped as `\\`.

**Common issues:**

| Symptom | Likely cause |
|---|---|
| Connector not shown in "+" menu | Path to `dist/index.js` is wrong, or `npm run build` was not run |

## Remote Deployment (Vercel)

Deploy to Vercel to use the esa MCP server from the **browser version of Claude** (claude.ai) via Add connector.

### Step 1 — Push to GitHub

Make sure your latest code is pushed to GitHub. The `api/mcp.ts` and `vercel.json` files must be included.

### Step 2 — Import the Project in Vercel

1. Go to [vercel.com](https://vercel.com) and sign in
2. Click **"Add New… → Project"**
3. Select your GitHub repository
4. Leave the build settings at their defaults — Vercel auto-detects Node.js projects
5. Click **"Deploy"**

### Step 3 — Set Environment Variables

In your Vercel project dashboard, go to **Settings → Environment Variables** and add:

| Variable | Value |
|---|---|
| `ESA_ACCESS_TOKEN` | Your esa Personal Access Token (PAT v1) |
| `ESA_TEAM_NAME` | Your esa team name (the `xxx` part of `xxx.esa.io`) |
| `MCP_AUTH_TOKEN` | Any strong random string you choose (this is your private key to protect the endpoint) |

> To generate a secure token: `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"` or use any password generator.

After adding variables, click **"Redeploy"** from the Deployments tab to apply them.

### Step 4 — Find Your Endpoint URL

Your MCP endpoint is:
```
https://<your-project-name>.vercel.app/api/mcp
```

You can see your project URL on the Vercel dashboard.

### Step 5 — Add as a Connector in Claude

1. Open [claude.ai](https://claude.ai) in your browser
2. Click your profile icon → **Settings**
3. Go to **"Integrations"** (or **"Connectors"**)
4. Click **"Add connector"** → **"Add a custom integration"**
5. Enter your endpoint URL: `https://<your-project-name>.vercel.app/api/mcp`
6. Under authentication, select **"API Key"** or **"Bearer token"** and enter your `MCP_AUTH_TOKEN` value
7. Save

Once added, the esa tools will appear in your Claude chat sessions.

## Development

### Why a Build Step Is Needed

The source code is written in **TypeScript** (`src/index.ts`). Node.js cannot run TypeScript directly — it only understands JavaScript. So the code must be **compiled** first.

```
src/index.ts  ──(tsc compiles)──▶  dist/index.js  ──(node runs)──▶  MCP Server
  TypeScript                          JavaScript
  (you edit)                      (Claude Desktop uses this)
```

`npm run build` runs the TypeScript compiler (`tsc`) and produces `dist/index.js`.

**After editing `src/index.ts`, you must always run `npm run build` before the changes take effect in Claude Desktop.**

---

### Development Mode vs. Build

| | `npm run dev` | `npm run build` |
|---|---|---|
| Purpose | Testing while developing | Deploying for Claude Desktop |
| How it works | `tsx` converts TypeScript on-the-fly and runs it in your terminal | `tsc` compiles `src/index.ts` → `dist/index.js` |
| Produces `dist/index.js`? | No | Yes |
| Usable by Claude Desktop? | No — Claude Desktop cannot connect to a terminal process | Yes |
| After code change | Just restart the command | Run `npm run build` again |

In short: use `npm run dev` while writing code, then run `npm run build` when you want Claude Desktop to pick up the changes.

---

### Adding a New Tool

1. Open [src/index.ts](src/index.ts)
2. Add a tool definition in the `ListToolsRequestSchema` handler (around line 52)
3. Add a case for the tool in the `CallToolRequestSchema` handler (around line 194)
4. Run `npm run build` to compile, then restart Claude Desktop

### Build

```bash
npm run build
```

Compiles TypeScript to `dist/index.js`.

# esa MCP サーバー

Claude から直接 esa の記事を作成・編集・検索・管理できる MCP（Model Context Protocol）サーバーです。ブラウザを開かなくても esa を操作できます。

2つの利用モードに対応しています:

| モード | クライアント | 通信方式 |
|---|---|---|
| **ローカル** | Claude Desktop（デスクトップアプリ） | stdio（プロセス間通信） |
| **リモート** | Claude（ブラウザ版 — claude.ai） | HTTP（Vercel 上の Streamable HTTP） |

## できること

| ツール名 | 説明 |
|---|---|
| `esa_list_posts` | 記事一覧の取得・検索（カテゴリ、キーワード、タグ、WIP状態などで絞り込み可） |
| `esa_get_post` | 記事番号で特定の記事を取得 |
| `esa_create_post` | 新規記事の作成（カテゴリ・タグ・WIPフラグ・変更メッセージ指定可） |
| `esa_update_post` | 既存記事の更新 |
| `esa_delete_post` | 記事の削除 |
| `esa_get_members` | チームメンバー一覧の取得 |

---

## アーキテクチャ

### ローカルモード（Claude Desktop）

```
あなた（チャットに入力）
      │
      ▼
┌──────────────────────────┐

│                          │  呼び出すツールを決定する
└────────────┬─────────────┘

┌──────────────────────────┐
│      esa MCP サーバー     │  ◀──  このリポジトリ（src/index.ts）
│      dist/index.js       │  ツール呼び出しを受け取り、
│    Node.js で動作        │  esa API を叩いて結果を返す
└────────────┬─────────────┘
             │ HTTPS（esa REST API）
             ▼
┌──────────────────────────┐
│      あなたの esa チーム  │  xxx.esa.io
└──────────────────────────┘
```

### リモートモード（ブラウザ版 Claude via Vercel）

```
あなた（claude.ai でチャットに入力）
      │
      ▼
┌──────────────────────────┐
│     Claude（ブラウザ版）  │  claude.ai
│    （Claude AI モデル）   │  メッセージを理解し、
│                          │  呼び出すツールを決定する
└────────────┬─────────────┘
             │ MCP Streamable HTTP
             │ Authorization: Bearer <MCP_AUTH_TOKEN>
             ▼
┌──────────────────────────┐
│  Vercel サーバーレス関数  │  ◀──  このリポジトリ（api/mcp.ts）
│  https://xxx.vercel.app  │  Bearer トークンを検証し、
│       /api/mcp           │  esa API を叩いて結果を返す
└────────────┬─────────────┘
             │ HTTPS（esa REST API）
             ▼
┌──────────────────────────┐
│      あなたの esa チーム  │  xxx.esa.io
└──────────────────────────┘
```

**処理の流れの例 — 「最近の記事を5件見せて」の場合:**

1. あなたが Claude にメッセージを入力する
2. Claude（AI）が意図を理解し、`esa_list_posts` ツールを呼び出す
3. MCP サーバーが呼び出しを受け取り、esa API に GET リクエストを送る
4. esa が記事データを JSON で返す
5. MCP サーバーがそれを Claude に渡す
6. Claude が読みやすい形に整形して表示する

### 各ファイルの役割

```
esa-mcp-server/
│
├── src/
│   ├── server.ts       ← 共有 MCP サーバーファクトリ（全ツール定義と
│   │                      esa API 呼び出し処理）。両モードで使われる。
│   └── index.ts        ← Claude Desktop 用の stdio エントリーポイント。
│                          環境変数を読み込み、StdioServerTransport で接続。
│
├── api/
│   └── mcp.ts          ← ブラウザ版 Claude 用の Vercel サーバーレス関数。
│                          HTTP リクエストを受け取り、Bearer トークン認証を行い、
│                          StreamableHTTPServerTransport で接続。
│
├── dist/               ← コンパイル済みファイル（npm run build で自動生成）
│   ├── server.js          直接編集しないこと。
│   └── index.js           Claude Desktop は dist/index.js を実行する。
│
├── vercel.json         ← Vercel デプロイ設定
├── package.json        ← プロジェクトの設定ファイル
├── tsconfig.json       ← TypeScript コンパイラの設定
└── .gitignore          ← node_modules/、dist/ を git から除外する設定
```

### `node_modules/` と `dist/` とは？

この2つのディレクトリは**リポジトリには含まれていない**ため、クローン後に手元で生成する必要があります:

| ディレクトリ | 内容 | 生成コマンド |
|---|---|---|
| `node_modules/` | このプロジェクトが使うライブラリ群（MCP SDK など） | `npm install` |
| `dist/` | Claude Desktop が実行する JavaScript ファイル | `npm run build` |

## リモートデプロイ（Vercel）

Vercel にデプロイすることで、**ブラウザ版 Claude**（claude.ai）の Add connector から esa ツールを使えるようになります。

### ステップ 1 — GitHub にプッシュ

`api/mcp.ts` と `vercel.json` を含む最新のコードが GitHub にプッシュされている状態にしてください。

### ステップ 2 — Vercel にプロジェクトをインポート

1. [vercel.com](https://vercel.com) にサインインする
2. **"Add New… → Project"** をクリック
3. GitHub リポジトリを選択する
4. ビルド設定はデフォルトのままでOK（Vercel が Node.js プロジェクトを自動検出する）
5. **"Deploy"** をクリック

### ステップ 3 — 環境変数を設定

Vercel のプロジェクトダッシュボードで **Settings → Environment Variables** を開き、以下を追加します:

| 変数名 | 値 |
|---|---|
| `ESA_ACCESS_TOKEN` | esa の Personal Access Token（PAT v1） |
| `ESA_TEAM_NAME` | esa のチーム名（`xxx.esa.io` の `xxx` 部分） |
| `MCP_AUTH_TOKEN` | 自分で決めた任意のランダムな文字列（エンドポイントを保護するための秘密のキー） |

> セキュアなトークンを生成するコマンド: `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`

環境変数を追加したら、Deployments タブから **"Redeploy"** をクリックして反映させます。

### ステップ 4 — エンドポイント URL を確認

MCP エンドポイントの URL は以下の形式になります:
```
https://<プロジェクト名>.vercel.app/api/mcp
```

Vercel ダッシュボードでプロジェクトの URL を確認してください。

### ステップ 5 — Claude に Connector として追加

1. ブラウザで [claude.ai](https://claude.ai) を開く
2. プロフィールアイコン → **Settings** をクリック
3. **"Integrations"**（または **"Connectors"**）を開く
4. **"Add connector"** → **"Add a custom integration"** をクリック
5. エンドポイント URL を入力: `https://<プロジェクト名>.vercel.app/api/mcp`
6. 認証設定で **"API Key"** または **"Bearer token"** を選択し、`MCP_AUTH_TOKEN` の値を入力
7. 保存

追加後は、Claude のチャット画面で esa ツールが使えるようになります。

## 開発（機能の追加・拡張）

### なぜビルドが必要なのか

ソースコードは **TypeScript**（`src/index.ts`）で書かれています。Node.js は TypeScript をそのまま実行できず、JavaScript しか理解できません。そのため、コードを**コンパイル（変換）**する必要があります。

```
src/index.ts  ──（tsc がコンパイル）──▶  dist/index.js  ──（node が実行）──▶  MCP サーバー
  TypeScript                               JavaScript
  （編集するファイル）                  （Claude Desktop が使うファイル）
```

`npm run build` を実行すると TypeScript コンパイラ（`tsc`）が動き、`dist/index.js` が生成されます。

**`src/index.ts` を編集したら、必ず `npm run build` を実行してください。それをしないと Claude Desktop に変更が反映されません。**

---

### 開発モードとビルドの違い

| | `npm run dev` | `npm run build` |
|---|---|---|
| 用途 | 開発中の動作確認 | Claude Desktop に接続して使う |
| 仕組み | `tsx` が TypeScript をその場で変換してターミナルで実行 | `tsc` が `src/index.ts` → `dist/index.js` にコンパイル |
| `dist/index.js` を生成する？ | しない | する |
| Claude Desktop から使える？ | 使えない（ターミナルのプロセスには接続できない） | 使える |
| コード変更後 | コマンドを再起動するだけ | `npm run build` を再実行する必要あり |

まとめると：**コードを書いている間は `npm run dev`、Claude Desktop に反映させたいときは `npm run build`** という使い分けです。

---

### ツールを追加する方法

1. [src/index.ts](src/index.ts) を開く
2. `ListToolsRequestSchema` ハンドラー（約 52 行目付近）にツール定義を追加する
3. `CallToolRequestSchema` ハンドラー（約 194 行目付近）に対応する `case` を追加する
4. `npm run build` を実行してコンパイルし、Claude Desktop を再起動する

### ビルド

```bash
npm run build
```

TypeScript を `dist/index.js` にコンパイルします。

## 動作要件

- Node.js v18 以上
- Claude Desktop（ローカル stdio モード）または claude.ai アカウント（リモート HTTP モード）
- esa チームアカウントおよび `read` と `write` スコープを持つ Personal Access Token v1（PAT v1）
- Vercel デプロイの場合: Vercel アカウント（無料の Hobby プランで動作します）

### 環境変数

| 変数名 | 必要なモード | 説明 |
|---|---|---|
| `ESA_ACCESS_TOKEN` | 両モード | esa Personal Access Token v1 |
| `ESA_TEAM_NAME` | 両モード | esa チーム名（`xxx.esa.io` の `xxx` 部分） |
| `MCP_AUTH_TOKEN` | リモート（Vercel）のみ | HTTP エンドポイントを保護する Bearer トークン |
