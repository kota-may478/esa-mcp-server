# esa MCP Server

An MCP (Model Context Protocol) server that lets you create, edit, search, and manage esa posts directly from Claude Desktop — no browser required.

## What You Can Do

| Tool | Description |
|---|---|
| `esa_list_posts` | List and search posts (filter by category, keyword, tag, WIP status, etc.) |
| `esa_get_post` | Retrieve a specific post by post number |
| `esa_create_post` | Create a new post (with optional category, tags, WIP flag, and change message) |
| `esa_update_post` | Update an existing post |
| `esa_delete_post` | Delete a post |
| `esa_get_members` | List all team members |

---

## Architecture

### How Everything Fits Together

```
You (type in chat)
      │
      ▼
┌──────────────────────────┐
│      Claude Desktop      │  AI assistant app on your PC
│    (Claude AI model)     │  understands your message and
│                          │  decides which tool to call
└────────────┬─────────────┘
             │ MCP protocol (stdio)
             │ launches and talks to ↓
             ▼
┌──────────────────────────┐
│      esa MCP Server      │  ◀──  this repository
│      dist/index.js       │  receives the tool call,
│    running on Node.js    │  calls the esa API, returns results
└────────────┬─────────────┘
             │ HTTPS  (esa REST API)
             ▼
┌──────────────────────────┐
│         esa API          │  api.esa.io/v1/...
│                          │  authenticates with your token
└────────────┬─────────────┘
             │
             ▼
┌──────────────────────────┐
│      Your esa Team       │  xxx.esa.io
│   (posts, members, ...)  │
└──────────────────────────┘
```

**Flow example — "Show me the 5 most recent posts":**

1. You type the message in Claude Desktop
2. Claude (AI) understands the intent and calls the `esa_list_posts` tool
3. The MCP server receives the call and sends a GET request to the esa API
4. esa returns the post data as JSON
5. The MCP server passes it back to Claude
6. Claude formats and displays it as a readable response

### Role of Each File

```
esa-mcp-server/
│
├── src/
│   └── index.ts        ← THE source code you edit (TypeScript)
│                          Defines all 6 tools and their esa API calls.
│                          Node.js cannot run this file directly.
│
├── dist/
│   └── index.js        ← Compiled output (JavaScript)
│                          Auto-generated by `npm run build`.
│                          This is what Claude Desktop actually runs.
│                          Do not edit this file directly.
│
├── package.json        ← Project metadata and npm script definitions
│                          (build / dev / start commands are defined here)
│
├── tsconfig.json       ← TypeScript compiler settings
│                          (output directory, JS version, strictness, etc.)
│
└── .gitignore          ← Tells git which files not to track
                           (node_modules/, dist/ are excluded)
```

### What Are `node_modules/` and `dist/`?

These two directories are **not stored in the repository** and must be generated locally:

| Directory | What it contains | How to generate |
|---|---|---|
| `node_modules/` | Libraries this project depends on (MCP SDK, etc.) | `npm install` |
| `dist/` | The compiled JavaScript file Claude Desktop runs | `npm run build` |

## Setup

### Step 1 — Get an esa Access Token

1. Log in to esa and open `https://[your-team-name].esa.io/user/applications`
2. Under **Personal Access Tokens**, click **"Generate new token"**
3. Select both **`read`** and **`write`** scopes, then generate the token
4. Copy and save the token somewhere safe — it is shown only once

> **Token type:** Use **Personal Access Token v1 (PAT v1)**.

> **Team name:** The team name is the `xxx` part of `xxx.esa.io`.

### Step 5 — Edit the Claude Desktop Configuration File

**File location:**
- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

**To open on Windows (PowerShell):**
```powershell
notepad "$env:APPDATA\Claude\claude_desktop_config.json"
```

Add the following to the file (replace `YOUR_*` with your actual values):

```json
{
  "mcpServers": {
    "esa": {
      "command": "node",
      "args": [
        "/absolute/path/to/esa-mcp-server/dist/index.js"
      ],
      "env": {
        "ESA_ACCESS_TOKEN": "YOUR_ESA_ACCESS_TOKEN",
        "ESA_TEAM_NAME": "YOUR_TEAM_NAME"
      }
    }
  }
}
```

**Path examples:**

macOS:
```json
"/Users/yourname/projects/esa-mcp-server/dist/index.js"
```

Windows:
```json
"C:\\Users\\yourname\\projects\\esa-mcp-server\\dist\\index.js"
```

> **Always use absolute paths.** Relative paths and `~` are not supported.

> On Windows, backslashes in JSON must be escaped as `\\`.

## Development

### Why a Build Step Is Needed

The source code is written in **TypeScript** (`src/index.ts`). Node.js cannot run TypeScript directly — it only understands JavaScript. So the code must be **compiled** first.

```
src/index.ts  ──(tsc compiles)──▶  dist/index.js  ──(node runs)──▶  MCP Server
  TypeScript                          JavaScript
  (you edit)                      (Claude Desktop uses this)
```

`npm run build` runs the TypeScript compiler (`tsc`) and produces `dist/index.js`.

**After editing `src/index.ts`, you must always run `npm run build` before the changes take effect in Claude Desktop.**

---

### Development Mode vs. Build

| | `npm run dev` | `npm run build` |
|---|---|---|
| Purpose | Testing while developing | Deploying for Claude Desktop |
| How it works | `tsx` converts TypeScript on-the-fly and runs it in your terminal | `tsc` compiles `src/index.ts` → `dist/index.js` |
| Produces `dist/index.js`? | No | Yes |
| Usable by Claude Desktop? | No — Claude Desktop cannot connect to a terminal process | Yes |
| After code change | Just restart the command | Run `npm run build` again |

In short: use `npm run dev` while writing code, then run `npm run build` when you want Claude Desktop to pick up the changes.

### Adding a New Tool

1. Open [src/index.ts](src/index.ts)
2. Add a tool definition in the `ListToolsRequestSchema` handler (around line 52)
3. Add a case for the tool in the `CallToolRequestSchema` handler (around line 194)
4. Run `npm run build` to compile, then restart Claude Desktop

### Build

```bash
npm run build
```

Compiles TypeScript to `dist/index.js`.

# esa MCP サーバー

Claude Desktop から直接 esa の記事を作成・編集・検索・管理できる MCP（Model Context Protocol）サーバーです。ブラウザを開かなくても esa を操作できます。

## できること

| ツール名 | 説明 |
|---|---|
| `esa_list_posts` | 記事一覧の取得・検索（カテゴリ、キーワード、タグ、WIP状態などで絞り込み可） |
| `esa_get_post` | 記事番号で特定の記事を取得 |
| `esa_create_post` | 新規記事の作成（カテゴリ・タグ・WIPフラグ・変更メッセージ指定可） |
| `esa_update_post` | 既存記事の更新 |
| `esa_delete_post` | 記事の削除 |
| `esa_get_members` | チームメンバー一覧の取得 |

**処理の流れの例 — 「最近の記事を5件見せて」の場合:**

5. MCP サーバーがそれを Claude に渡す
6. Claude が読みやすい形に整形して表示する

## 開発（機能の追加・拡張）

### なぜビルドが必要なのか

ソースコードは **TypeScript**（`src/index.ts`）で書かれています。Node.js は TypeScript をそのまま実行できず、JavaScript しか理解できません。そのため、コードを**コンパイル（変換）**する必要があります。

```
src/index.ts  ──（tsc がコンパイル）──▶  dist/index.js  ──（node が実行）──▶  MCP サーバー
  TypeScript                               JavaScript
  （編集するファイル）                  （Claude Desktop が使うファイル）
```

`npm run build` を実行すると TypeScript コンパイラ（`tsc`）が動き、`dist/index.js` が生成されます。

**`src/index.ts` を編集したら、必ず `npm run build` を実行してください。それをしないと Claude Desktop に変更が反映されません。**

---

### 開発モードとビルドの違い

| | `npm run dev` | `npm run build` |
|---|---|---|
| 用途 | 開発中の動作確認 | Claude Desktop に接続して使う |
| 仕組み | `tsx` が TypeScript をその場で変換してターミナルで実行 | `tsc` が `src/index.ts` → `dist/index.js` にコンパイル |
| `dist/index.js` を生成する？ | しない | する |
| Claude Desktop から使える？ | 使えない（ターミナルのプロセスには接続できない） | 使える |
| コード変更後 | コマンドを再起動するだけ | `npm run build` を再実行する必要あり |

まとめると：**コードを書いている間は `npm run dev`、Claude Desktop に反映させたいときは `npm run build`** という使い分けです。

---

### ツールを追加する方法

1. [src/index.ts](src/index.ts) を開く
2. `ListToolsRequestSchema` ハンドラー（約 52 行目付近）にツール定義を追加する
3. `CallToolRequestSchema` ハンドラー（約 194 行目付近）に対応する `case` を追加する
4. `npm run build` を実行してコンパイルし、Claude Desktop を再起動する

### ビルド

```bash
npm run build
```

TypeScript を `dist/index.js` にコンパイルします。
