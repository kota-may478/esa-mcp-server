# esa MCP Server

An MCP (Model Context Protocol) server that lets you create, edit, search, and manage esa posts directly from Claude. After finishing your work in Claude, you can skip the usual cycle of "ask Claude to summarize → open esa → create a post → copy and paste" and update esa without ever leaving Claude.

Two usage modes are supported:

| Mode | Client | Transport |
|---|---|---|
| **Local** | Claude Desktop (desktop app) | stdio (process communication) |
| **Remote** | Claude (browser — claude.ai) | HTTP (Streamable HTTP on Vercel) |

## What You Can Do

| Tool | Description |
|---|---|
| `esa_list_posts` | List and search posts (filter by category, keyword, tag, WIP status, etc.) |
| `esa_get_post` | Retrieve a specific post by post number |
| `esa_create_post` | Create a new post (with optional category, tags, WIP flag, and change message) |
| `esa_update_post` | Update an existing post |
| `esa_delete_post` | Delete a post |
| `esa_get_members` | List all team members |

## Architecture

### Local Mode (Claude Desktop)

```
You (type in chat)
      │
      ▼
┌──────────────────────────┐
│      Claude Desktop      │  AI assistant app on your PC
│    (Claude AI model)     │  understands your message and
│                          │  decides which tool to call
└────────────┬─────────────┘
             │ MCP protocol (stdio)
             │ launches and talks to ↓
             ▼
┌──────────────────────────┐
│      esa MCP Server      │  ◀──  this repository (src/index.ts)
│      dist/index.js       │  receives the tool call,
│    running on Node.js    │  calls the esa API, returns results
└────────────┬─────────────┘
             │ HTTPS  (esa REST API)
             ▼
┌──────────────────────────┐
│      Your esa Team       │  xxx.esa.io
└──────────────────────────┘
```

### Remote Mode (Browser Claude via Vercel)

```
You (type in chat at claude.ai)
      │
      ▼
┌──────────────────────────┐
│     Claude (browser)     │  claude.ai
│    (Claude AI model)     │  understands your message and
│                          │  decides which tool to call
└────────────┬─────────────┘
             │ MCP Streamable HTTP
             │ https://xxx.vercel.app/api/mcp?token=<MCP_AUTH_TOKEN>
             ▼
┌──────────────────────────┐
│   Vercel Serverless Fn   │  ◀──  this repository (api/mcp.ts)
│  https://xxx.vercel.app  │  verifies token in URL query param,
│       /api/mcp           │  calls the esa API, returns results
└────────────┬─────────────┘
             │ HTTPS  (esa REST API)
             ▼
┌──────────────────────────┐
│      Your esa Team       │  xxx.esa.io
└──────────────────────────┘
```

**Flow example — "Show me the 5 most recent posts":**

1. You type the message in Claude
2. Claude (AI) understands the intent and calls the `esa_list_posts` tool
3. The MCP server receives the call and sends a GET request to the esa API
4. esa returns the post data as JSON
5. The MCP server passes it back to Claude
6. Claude formats and displays it as a readable response

### Role of Each File

```
esa-mcp-server/
│
├── src/
│   ├── server.ts       ← Shared MCP server factory (all tool definitions
│   │                      and esa API calls). Used by both modes.
│   └── index.ts        ← stdio entry point for Claude Desktop.
│                          Reads env vars, creates the server, connects
│                          via StdioServerTransport.
│
├── api/
│   └── mcp.ts          ← Vercel serverless function for browser Claude.
│                          Handles HTTP requests, verifies auth token
│                          (query param ?token= or Bearer header),
│                          creates the server, connects via
│                          StreamableHTTPServerTransport.
│
├── dist/               ← Compiled output (auto-generated by npm run build)
│   ├── server.js          Do not edit these files directly.
│   └── index.js           Claude Desktop runs dist/index.js.
│
├── vercel.json         ← Vercel deployment configuration
├── package.json        ← Project metadata and npm script definitions
├── tsconfig.json       ← TypeScript compiler settings
└── .gitignore          ← Excludes node_modules/ and dist/ from git
```

### What Are `node_modules/` and `dist/`?

These two directories are **not stored in the repository** and must be generated locally:

| Directory | What it contains | How to generate |
|---|---|---|
| `node_modules/` | Libraries this project depends on (MCP SDK, etc.) | `npm install` |
| `dist/` | The compiled JavaScript file Claude Desktop runs | `npm run build` |

## Setup: Local Mode — Claude Desktop

### Step 1 — Get an esa Access Token

1. Log in to esa and open `https://[your-team-name].esa.io/user/applications`
2. Under **Personal Access Tokens**, click **"Generate new token"**
3. Select both **`read`** and **`write`** scopes, then generate the token
4. Copy and save the token somewhere safe — it is shown only once

> **Token type:** Use **Personal Access Token v1 (PAT v1)**.

> **Team name:** The team name is the `xxx` part of `xxx.esa.io`.

---

### Step 2 — Install Node.js

**macOS**

```bash
# Using Homebrew
brew install node

# Or download the .pkg installer from https://nodejs.org
```

**Windows**

Download the **.msi installer** from [https://nodejs.org](https://nodejs.org) and run it.

After installation, open **PowerShell** and check the version:

```powershell
node --version   # v18 or later required
npm --version
```

> **If npm commands fail in PowerShell** (e.g. "running scripts is disabled on this system"), run this once:

> Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

Or download and extract the ZIP, then navigate to the folder.

### Step 4 — Install Dependencies and Build

```bash

```

---

### Step 5 — Edit the Claude Desktop Configuration File

**File location:**
- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

**To open on Windows (PowerShell):**
```powershell
start "$env:APPDATA\Claude\claude_desktop_config.json"
```
This opens the file with your default text editor (Notepad, VS Code, etc.).

**If the file does not exist (first MCP server):** Create it as a new file and paste in the entire block below.

**If the file already exists (other MCP servers are configured):** Add only the `"esa": { ... }` block inside the existing `"mcpServers"` object. Do not replace the whole file — that would remove your other MCP servers.

```json
{
  "mcpServers": {
    "esa": {
      "command": "node",
      "args": [
        "/absolute/path/to/esa-mcp-server/dist/index.js"
      ],
      "env": {
        "ESA_ACCESS_TOKEN": "YOUR_ESA_ACCESS_TOKEN",
        "ESA_TEAM_NAME": "YOUR_TEAM_NAME"
      }
    }
  }
}
```

Replace `YOUR_*` with your actual values. Path examples:

macOS:
```json
"/Users/yourname/projects/esa-mcp-server/dist/index.js"
```

Windows:
```json
"C:\\Users\\yourname\\projects\\esa-mcp-server\\dist\\index.js"
```

> **Always use absolute paths.** Relative paths and `~` are not supported.

> On Windows, backslashes in JSON must be escaped as `\\`.

## Setup: Remote Mode — claude.ai (via Vercel)

Deploy your own instance to Vercel to use the esa MCP server from the **browser version of Claude** (claude.ai) via Add connector.

Each person who wants to use this remotely must deploy their own instance — this keeps your esa credentials and endpoint private.

### Step 1 — Fork the Repository

Fork this repository to your own GitHub account so you can connect it to Vercel:

1. Click **"Fork"** at the top-right of this GitHub page
2. Select your own GitHub account as the destination

> If you prefer not to fork, you can clone the repository locally and push it to a new private repository on your account instead.

### Step 2 — Import the Project in Vercel

1. Go to [vercel.com](https://vercel.com) and sign in (create a free account if needed)
2. Click **"Add New… → Project"**
3. Select your forked repository
4. Leave the build settings at their defaults
5. Click **"Deploy"**

The initial deploy will fail or show an error — that is expected at this point because the environment variables have not been set yet. Continue to the next step.

### Step 3 — Set Environment Variables

In your Vercel project dashboard, go to **Settings → Environment Variables** and add these three variables:

| Variable | Value |
|---|---|
| `ESA_ACCESS_TOKEN` | Your esa Personal Access Token (PAT v1) — see [Setup Step 1](#step-1--get-an-esa-access-token) |
| `ESA_TEAM_NAME` | Your esa team name (the `xxx` part of `xxx.esa.io`) |
| `MCP_AUTH_TOKEN` | A secret string **you generate yourself** — this protects your endpoint from unauthorized access |

**To generate a secure `MCP_AUTH_TOKEN`:**
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```
Copy the output and use it as the value. Save it somewhere safe — you will need it in Step 5.

After adding all three variables, go to the **Deployments** tab and click **"Redeploy"** to apply them.

### Step 4 — Find Your Endpoint URL

After a successful deployment, your MCP endpoint is:
```
https://<your-project-name>.vercel.app/api/mcp
```

You can find your exact URL on the Vercel project dashboard.

### Step 5 — Add as a Connector in Claude

1. Open [claude.ai](https://claude.ai) in your browser
2. Click your profile icon → **Settings**
3. Go to **"Integrations"** (or **"Connectors"**)
4. Click **"Add connector"** → **"Add a custom integration"**
5. Enter your endpoint URL **with the token embedded as a query parameter**:
   ```
   https://<your-project-name>.vercel.app/api/mcp?token=<MCP_AUTH_TOKEN>
   ```
   Replace `<MCP_AUTH_TOKEN>` with the secret string you generated in Step 3.
6. Leave the OAuth fields blank (not needed)
7. Save

### Verifying and Using the Connector

Once added, open a new chat in Claude and click the **"+"** (or the tools icon) at the bottom of the chat input area. You should see `esa-mcp-server` listed there. Enable it and then talk to Claude naturally:

```
Show me the 5 most recently updated posts.
```

> **Tool approval prompt:** The first time a tool is called, Claude may show a confirmation dialog asking for permission to run it. Click **"Allow"** to proceed.

### When You Update the Code

If you add new tools or make changes to the code:

1. Commit and push your changes to your forked GitHub repository
2. Vercel will automatically redeploy — no manual steps needed

## Development

### Why a Build Step Is Needed

The source code is written in **TypeScript** (`src/index.ts`). Node.js cannot run TypeScript directly — it only understands JavaScript. So the code must be **compiled** first.

```
src/index.ts  ──(tsc compiles)──▶  dist/index.js  ──(node runs)──▶  MCP Server
  TypeScript                          JavaScript
  (you edit)                      (Claude Desktop uses this)
```

`npm run build` runs the TypeScript compiler (`tsc`) and produces `dist/index.js`.

**After editing `src/server.ts` or `src/index.ts`, you must always run `npm run build` before the changes take effect in Claude Desktop.**

---

### Development Mode vs. Build

| | `npm run dev` | `npm run build` |
|---|---|---|
| Purpose | Testing while developing | Deploying for Claude Desktop |
| How it works | `tsx` converts TypeScript on-the-fly and runs it in your terminal | `tsc` compiles `src/index.ts` → `dist/index.js` |
| Produces `dist/index.js`? | No | Yes |
| Usable by Claude Desktop? | No — Claude Desktop cannot connect to a terminal process | Yes |
| After code change | Just restart the command | Run `npm run build` again |

In short: use `npm run dev` while writing code, then run `npm run build` when you want Claude Desktop to pick up the changes.

---

### Adding a New Tool

1. Open [src/server.ts](src/server.ts)
2. Add a tool definition in the `ListToolsRequestSchema` handler (around line 42)
3. Add a case for the tool in the `CallToolRequestSchema` handler (around line 184)
4. Run `npm run build` to compile, then restart Claude Desktop

### Build

```bash
npm run build
```

Compiles TypeScript to `dist/index.js`.

# esa MCP サーバー

Claude から直接 esa の記事を作成・編集・検索・管理できる MCP（Model Context Protocol）サーバーです。Claude で作業をした後に「Claude に情報をまとめてもらう → esa のページを開く → 記事を作成してコピペする」という手順を踏まずに、快適に esa を更新できます。

2つの利用モードに対応しています:

| モード | クライアント | 通信方式 |
|---|---|---|
| **ローカル** | Claude Desktop（デスクトップアプリ） | stdio（プロセス間通信） |
| **リモート** | Claude（ブラウザ版 — claude.ai） | HTTP（Vercel 上の Streamable HTTP） |

## できること

| ツール名 | 説明 |
|---|---|
| `esa_list_posts` | 記事一覧の取得・検索（カテゴリ、キーワード、タグ、WIP状態などで絞り込み可） |
| `esa_get_post` | 記事番号で特定の記事を取得 |
| `esa_create_post` | 新規記事の作成（カテゴリ・タグ・WIPフラグ・変更メッセージ指定可） |
| `esa_update_post` | 既存記事の更新 |
| `esa_delete_post` | 記事の削除 |
| `esa_get_members` | チームメンバー一覧の取得 |

## アーキテクチャ

### ローカルモード（Claude Desktop）

```
あなた（チャットに入力）
      │
      ▼
┌──────────────────────────┐
│      Claude Desktop      │  PC 上の AI アシスタントアプリ
│    （Claude AI モデル）   │  メッセージを理解し、
│                          │  呼び出すツールを決定する
└────────────┬─────────────┘
             │ MCP プロトコル（stdio）
             │ 起動・通信 ↓
             ▼
┌──────────────────────────┐
│      esa MCP サーバー     │  ◀──  このリポジトリ（src/index.ts）
│      dist/index.js       │  ツール呼び出しを受け取り、
│    Node.js で動作        │  esa API を叩いて結果を返す
└────────────┬─────────────┘
             │ HTTPS（esa REST API）
             ▼
┌──────────────────────────┐
│      あなたの esa チーム  │  xxx.esa.io
└──────────────────────────┘
```

### リモートモード（ブラウザ版 Claude via Vercel）

```
あなた（claude.ai でチャットに入力）
      │
      ▼
┌──────────────────────────┐
│     Claude（ブラウザ版）  │  claude.ai
│    （Claude AI モデル）   │  メッセージを理解し、
│                          │  呼び出すツールを決定する
└────────────┬─────────────┘
             │ MCP Streamable HTTP
             │ https://xxx.vercel.app/api/mcp?token=<MCP_AUTH_TOKEN>
             ▼
┌──────────────────────────┐
│  Vercel サーバーレス関数  │  ◀──  このリポジトリ（api/mcp.ts）
│  https://xxx.vercel.app  │  URL クエリパラメータのトークンを検証し、
│       /api/mcp           │  esa API を叩いて結果を返す
└────────────┬─────────────┘
             │ HTTPS（esa REST API）
             ▼
┌──────────────────────────┐
│      あなたの esa チーム  │  xxx.esa.io
└──────────────────────────┘
```

**処理の流れの例 — 「最近の記事を5件見せて」の場合:**

1. あなたが Claude にメッセージを入力する
2. Claude（AI）が意図を理解し、`esa_list_posts` ツールを呼び出す
3. MCP サーバーが呼び出しを受け取り、esa API に GET リクエストを送る
4. esa が記事データを JSON で返す
5. MCP サーバーがそれを Claude に渡す
6. Claude が読みやすい形に整形して表示する

### 各ファイルの役割

```
esa-mcp-server/
│
├── src/
│   ├── server.ts       ← 共有 MCP サーバーファクトリ（全ツール定義と
│   │                      esa API 呼び出し処理）。両モードで使われる。
│   └── index.ts        ← Claude Desktop 用の stdio エントリーポイント。
│                          環境変数を読み込み、StdioServerTransport で接続。
│
├── api/
│   └── mcp.ts          ← ブラウザ版 Claude 用の Vercel サーバーレス関数。
│                          HTTP リクエストを受け取り、トークン認証を行い
│                          （URL クエリパラメータ ?token= または Bearer ヘッダー）、
│                          StreamableHTTPServerTransport で接続。
│
├── dist/               ← コンパイル済みファイル（npm run build で自動生成）
│   ├── server.js          直接編集しないこと。
│   └── index.js           Claude Desktop は dist/index.js を実行する。
│
├── vercel.json         ← Vercel デプロイ設定
├── package.json        ← プロジェクトの設定ファイル
├── tsconfig.json       ← TypeScript コンパイラの設定
└── .gitignore          ← node_modules/、dist/ を git から除外する設定
```

### `node_modules/` と `dist/` とは？

この2つのディレクトリは**リポジトリには含まれていない**ため、クローン後に手元で生成する必要があります:

| ディレクトリ | 内容 | 生成コマンド |
|---|---|---|
| `node_modules/` | このプロジェクトが使うライブラリ群（MCP SDK など） | `npm install` |
| `dist/` | Claude Desktop が実行する JavaScript ファイル | `npm run build` |

### ステップ 5 — Claude Desktop の設定ファイルを編集

**ファイルの場所:**
- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

**Windows でテキストエディタで開くコマンド（PowerShell）:**
```powershell
start "$env:APPDATA\Claude\claude_desktop_config.json"
```
既定のテキストエディタ（メモ帳、VS Code など）でファイルが開きます。

**ファイルが存在しない場合（初めて MCP サーバーを設定する場合）:** 新規ファイルとして作成し、以下のブロック全体を貼り付けてください。

**ファイルがすでに存在する場合（他の MCP サーバーが設定済みの場合）:** 既存の `"mcpServers"` オブジェクトの中に `"esa": { ... }` のブロックだけを追加してください。ファイル全体を置き換えると他の MCP サーバーの設定が消えてしまいます。

```json
{
  "mcpServers": {
    "esa": {
      "command": "node",
      "args": [
        "/絶対パス/esa-mcp-server/dist/index.js"
      ],
      "env": {
        "ESA_ACCESS_TOKEN": "YOUR_ESA_ACCESS_TOKEN",
        "ESA_TEAM_NAME": "YOUR_TEAM_NAME"
      }
    }
  }
}
```

`YOUR_*` を実際の値に置き換えてください。パスの例:

macOS:
```json
"/Users/yourname/projects/esa-mcp-server/dist/index.js"
```

Windows:
```json
"C:\\Users\\yourname\\projects\\esa-mcp-server\\dist\\index.js"
```

> **パスは必ず絶対パスで指定してください。** 相対パスや `~` は使えません。

> Windows では JSON 内のバックスラッシュを `\\` とエスケープする必要があります。

### ステップ 6 — Claude Desktop を再起動

ウィンドウを閉じるだけでは不十分です。**完全に終了**してから再起動してください:

- **macOS:** `Cmd+Q` または Dock のアイコンを右クリック → 「終了」
- **Windows:** Claude Desktop のメニューバーで **「File → 終了」** をクリック（ウィンドウを閉じるだけではトレイに常駐し続けます）

その後、Claude Desktop を再起動してください。

## セットアップ — リモートモード — claude.ai （via Vercel）

Vercel に自分専用のインスタンスをデプロイすることで、**ブラウザ版 Claude**（claude.ai）の Add connector から esa ツールを使えるようになります。

リモートで利用したい人はそれぞれ自分専用のインスタンスをデプロイしてください。これにより、esa の認証情報やエンドポイントが他人に共有されることなく安全に使えます。

### ステップ 1 — リポジトリをフォーク

Vercel に接続するために、このリポジトリを自分の GitHub アカウントにフォークします:

1. この GitHub ページ右上の **"Fork"** をクリック
2. 自分の GitHub アカウントをフォーク先として選択

> フォークしたくない場合は、リポジトリをローカルにクローンして自分のアカウントの新しいリポジトリにプッシュしても構いません。

### ステップ 2 — Vercel にプロジェクトをインポート

1. [vercel.com](https://vercel.com) にサインインする（アカウントがなければ無料で作成）
2. **"Add New… → Project"** をクリック
3. フォークしたリポジトリを選択する
4. ビルド設定はデフォルトのままでOK
5. **"Deploy"** をクリック

最初のデプロイは失敗またはエラーになります — 環境変数をまだ設定していないので、この時点では正常な動作です。次のステップに進んでください。

### ステップ 3 — 環境変数を設定

Vercel のプロジェクトダッシュボードで **Settings → Environment Variables** を開き、以下の3つを追加します:

| 変数名 | 値 |
|---|---|
| `ESA_ACCESS_TOKEN` | esa の Personal Access Token（PAT v1）— [セットアップ ステップ 1](#ステップ-1--esa-アクセストークンの取得) を参照 |
| `ESA_TEAM_NAME` | esa のチーム名（`xxx.esa.io` の `xxx` 部分） |
| `MCP_AUTH_TOKEN` | **自分で生成する**秘密の文字列 — エンドポイントへの不正アクセスを防ぐための鍵 |

**セキュアな `MCP_AUTH_TOKEN` を生成するコマンド:**
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```
出力された文字列をコピーして値として使用してください。ステップ 5 で必要になるので安全な場所に保存しておいてください。

3つの変数を追加したら、**Deployments** タブで **"Redeploy"** をクリックして反映させます。

### ステップ 4 — エンドポイント URL を確認

デプロイが成功したら、MCP エンドポイントの URL は以下の形式になります:
```
https://<プロジェクト名>.vercel.app/api/mcp
```

正確な URL は Vercel プロジェクトダッシュボードで確認できます。

### ステップ 5 — Claude に Connector として追加

1. ブラウザで [claude.ai](https://claude.ai) を開く
2. プロフィールアイコン → **Settings** をクリック
3. **"Integrations"**（または **"Connectors"**）を開く
4. **"Add connector"** → **"Add a custom integration"** をクリック
5. エンドポイント URL を **トークンをクエリパラメータとして含む形式** で入力:
   ```
   https://<プロジェクト名>.vercel.app/api/mcp?token=<MCP_AUTH_TOKEN>
   ```
   `<MCP_AUTH_TOKEN>` はステップ 3 で生成した秘密の文字列に置き換えてください。
6. OAuth フィールドは空欄のままでよい（不要）
7. 保存

### Connector の確認と使い方

追加後、Claude の新しいチャットを開いてチャット入力欄下部の **「+」**（またはツールアイコン）をクリックします。そこに `esa-mcp-server` が表示されていれば成功です。有効にして、あとは自然に話しかけるだけです:

```
最近更新された記事を5件見せて。
```

> **ツール承認ダイアログ:** 初回のツール呼び出し時に、Claude から実行許可を求める確認ダイアログが表示されることがあります。**「Allow」** をクリックして進んでください。

### コードを更新したとき

新しいツールを追加したりコードを変更した場合:

1. 変更をコミットしてフォークした GitHub リポジトリにプッシュする
2. Vercel が自動で再デプロイします — 手動操作は不要

## 開発（機能の追加・拡張）

### なぜビルドが必要なのか

ソースコードは **TypeScript**（`src/index.ts`）で書かれています。Node.js は TypeScript をそのまま実行できず、JavaScript しか理解できません。そのため、コードを**コンパイル（変換）**する必要があります。

```
src/index.ts  ──（tsc がコンパイル）──▶  dist/index.js  ──（node が実行）──▶  MCP サーバー
  TypeScript                               JavaScript
  （編集するファイル）                  （Claude Desktop が使うファイル）
```

`npm run build` を実行すると TypeScript コンパイラ（`tsc`）が動き、`dist/index.js` が生成されます。

**`src/server.ts` または `src/index.ts` を編集したら、必ず `npm run build` を実行してください。それをしないと Claude Desktop に変更が反映されません。**

---

### 開発モードとビルドの違い

| | `npm run dev` | `npm run build` |
|---|---|---|
| 用途 | 開発中の動作確認 | Claude Desktop に接続して使う |
| 仕組み | `tsx` が TypeScript をその場で変換してターミナルで実行 | `tsc` が `src/index.ts` → `dist/index.js` にコンパイル |
| `dist/index.js` を生成する？ | しない | する |
| Claude Desktop から使える？ | 使えない（ターミナルのプロセスには接続できない） | 使える |
| コード変更後 | コマンドを再起動するだけ | `npm run build` を再実行する必要あり |

まとめると：**コードを書いている間は `npm run dev`、Claude Desktop に反映させたいときは `npm run build`** という使い分けです。

---

### ツールを追加する方法

1. [src/server.ts](src/server.ts) を開く
2. `ListToolsRequestSchema` ハンドラー（約 42 行目付近）にツール定義を追加する
3. `CallToolRequestSchema` ハンドラー（約 184 行目付近）に対応する `case` を追加する
4. `npm run build` を実行してコンパイルし、Claude Desktop を再起動する

### ビルド

```bash
npm run build
```

TypeScript を `dist/index.js` にコンパイルします。

## 動作要件

- Node.js v18 以上
- Claude Desktop（ローカル stdio モード）または claude.ai アカウント（リモート HTTP モード）
- esa チームアカウントおよび `read` と `write` スコープを持つ Personal Access Token v1（PAT v1）
- Vercel デプロイの場合: Vercel アカウント（無料の Hobby プランで動作します）

### 環境変数

| 変数名 | 必要なモード | 説明 |
|---|---|---|
| `ESA_ACCESS_TOKEN` | 両モード | esa Personal Access Token v1 |
| `ESA_TEAM_NAME` | 両モード | esa チーム名（`xxx.esa.io` の `xxx` 部分） |
| `MCP_AUTH_TOKEN` | リモート（Vercel）のみ | HTTP エンドポイントを保護する秘密のトークン。Connector 登録時は URL の `?token=` として渡す |
